# ForgeSpecs guide

A short, opinionated walkthrough of what ForgeSpecs is for and how to get value out of it quickly.

## What it is

ForgeSpecs is a self-hostable, AI-native platform for the canonical specs your engineering team should be writing anyway — Visions, PRDs, RFCs, ADRs, API specs, DB schemas, runbooks, task plans — kept linked, versioned, reviewable, and grounded in your actual code.

## How to read this guide

The sections are independent. Pick the one that matches what you're trying to do:

- **[Getting started](/guide/getting-started)** — sign up, workspaces, projects, the first doc.
- **[Import from a repo](/guide/import-from-repo)** — point ForgeSpecs at an existing codebase and slurp the docs out of it. Includes the GitHub PAT permissions section.
- **[Authoring](/guide/authoring)** — the editor, custom blocks (Mermaid, code, callouts), mentions, the slash menu.
- **[Reviews and status](/guide/reviews-and-status)** — moving a doc through DRAFT → REVIEW → APPROVED.
- **[AI and guidance](/guide/ai-and-guidance)** — the chat panel, what it can and can't do, how the guidance bot proposes actions.

If you just want one shortcut: hit `⌘J` anywhere in the app and ask the AI "what should I do next?". It reads your project state and tells you.
